# 기능요구정의서 (PRD)

## 제품명
뜨개 패턴 생성기

## 문서 목적
현재 서비스의 핵심 기능과 제약사항을 유지한 상태에서, UI/UX 개선용 디자인 작업을 진행하기 위한 기준 문서이다. 이 문서는 `stitch`에 전달할 요구사항 정의서로 사용한다.

## 프로젝트 목표
사용자가 뜨개 패턴의 기본 그리드를 설정하고, 요크 패턴 줄임 구조를 만들고, 편집 영역에서 색상/이미지/데이터를 활용해 패턴을 완성한 뒤, 반복/미리보기/다운로드까지 한 화면에서 수행할 수 있도록 한다.

## 디자인 작업 범위
- UI 개선
- UX 흐름 개선
- 정보 구조 재정리
- 편집 영역 중심 레이아웃 개선
- 입력 단계와 편집 단계의 명확한 구분

## 디자인 작업 제외 범위
- 기존 기능 삭제
- 계산 로직 변경
- 파일 포맷 변경
- 사용자 입력 규칙의 의미 변경
- 세로 나눗셈 계산 방식 변경

## 1. 제품 개요
이 서비스는 뜨개 패턴을 셀 기반 그리드로 생성하고 편집하는 도구이다. 사용자는 먼저 게이지와 그리드 크기를 설정하고, 필요 시 요크 줄임 설정으로 기본 패턴 구조를 만든다. 이후 편집 영역에서 색상 설정, 브러시 편집, 이미지/CSV/XLSX 불러오기를 통해 패턴을 수정한다. 완성된 결과는 반복 패턴으로 확장해 보고, 미리보기 및 CSV/XLSX/PDF로 다운로드할 수 있다.

## 2. 핵심 사용자
- 뜨개 패턴을 직접 설계하는 사용자
- 요크 형태 패턴을 빠르게 시각화하려는 사용자
- 이미지나 숫자 데이터를 뜨개용 셀 패턴으로 변환하려는 사용자

## 3. 핵심 사용자 흐름
1. 게이지 입력 또는 자동 게이지 선택
2. 전체 코수 / 시작 코수 / 전체 단수 입력
3. 요크 패턴 줄임 설정 입력
4. 그리드 생성
5. 편집 영역에서 색상 설정 및 패턴 수정
6. 필요 시 이미지 / CSV / XLSX 불러오기
7. 반복 패턴 확인
8. 미리보기 열기
9. CSV / XLSX / PDF 다운로드

## 4. 기능 요구사항

### 4-1. 게이지 설정
- 사용자는 `10cm당 코`, `10cm당 단`을 입력할 수 있어야 한다.
- `아직 게이지 없음` 체크 시 게이지 입력은 비활성화되어야 한다.
- 게이지가 없는 경우 자동 비율 `가로 1.5 / 세로 1`을 사용해야 한다.
- 게이지 입력 또는 자동 게이지 선택 후 셀 크기 정보가 계산되어 보여야 한다.

### 4-2. 그리드 셀 수 설정
- 사용자는 `전체 코수`, `시작 코수`, `전체 단수`를 입력할 수 있어야 한다.
- `전체 코수`, `전체 단수`는 전체 바탕 그리드 크기를 의미한다.
- `시작 코수`는 메인 실 색(1) 영역의 시작 폭을 의미한다.
- 이 단계 완료 후 다음 설정이 보여야 한다.

### 4-3. 요크 패턴 줄임 설정
- 사용자는 `시작단`, `늘림 마지막 단`, `총 늘림코`를 입력할 수 있어야 한다.
- 요크 자동 설정을 사용하지 않을 경우 체크박스로 끌 수 있어야 한다.
- 체크 시 관련 입력 UI는 숨겨져야 한다.
- 사용자가 값을 입력하기 전에는 요약 및 상세 계산값이 노출되지 않아야 한다.
- 입력 후 세로 나눗셈 계산 결과가 요약과 테이블로 보여야 한다.
- 상세 테이블은 `단 / 코 / 회` 형식이어야 한다.
- 요약은 세로 나눗셈 묶음 결과를 `단-코-회` 형식으로 보여야 한다.

### 4-4. 메인 색 미리 지정
- `메인 실 색 미리 지정하기` 체크 시 프리셋 색상 설정 UI가 열려야 한다.
- 사용자는 `셀 색(0)`과 `메인 실 색(1)`을 변경할 수 있어야 한다.

### 4-5. 그리드 생성
- 사용자가 입력한 값으로 기본 그리드를 생성해야 한다.
- 요크 설정 사용 시 메인 실 색(1) 기반 형태가 자동 생성되어야 한다.
- 요크 설정 미사용 시 빈 그리드가 생성되어야 한다.

### 4-6. 편집 영역
- 사용자는 셀 단위로 패턴을 수정할 수 있어야 한다.
- 좌클릭/드래그로 그리기 가능해야 한다.
- 우클릭/드래그로 지우기 가능해야 한다.
- 편집 영역은 서비스의 핵심 작업 공간이어야 한다.

### 4-7. 색상 설정
- 편집 영역에서 색 개수를 설정할 수 있어야 한다.
- `0`은 고정된 바탕값이며 사용자 배색 목록과 구분되어야 한다.
- `1`은 메인 실 색으로 강조되어야 한다.
- `1`부터 사용자가 원하는 색을 지정할 수 있어야 한다.
- 색 추가/삭제가 가능해야 한다.
- 선택한 색 삭제 시 번호가 자동 재정렬되어야 한다.

### 4-8. 파일 불러오기
- 편집 영역에서 파일 불러오기가 가능해야 한다.
- 지원 포맷:
  - 이미지
  - CSV
  - XLSX
- 파일 불러오기 시 현재 그리드와 요크 설정은 유지되어야 한다.
- 파일이 현재 그리드 기준 안에서만 적용되어야 한다.
- 파일이 더 크면 잘라서 적용해야 한다.

### 4-9. 이미지 불러오기
- 이미지는 현재 가로 셀 수 기준으로 픽셀 변환되어야 한다.
- 이미지의 색상은 현재 지원 색상 범위 내 숫자값으로 변환되어야 한다.
- 이미지 적용은 메인 실 색(1) 영역 기준으로 반영되어야 한다.
- 그리드 자체는 유지되고, 이미지가 그 위에 적용되는 방식이어야 한다.

### 4-10. CSV / XLSX 불러오기
- CSV/XLSX의 숫자값을 현재 그리드 안에 반영할 수 있어야 한다.
- 첫 시트 기준으로 XLSX를 읽을 수 있어야 한다.
- 잘못된 값은 안전하게 처리되어야 한다.
- 적용 시 현재 그리드 설정을 깨뜨리면 안 된다.

### 4-11. 반복 패턴
- 원본 패턴 생성 이후에만 반복 패턴 UI가 보여야 한다.
- 사용자는 X/Y 반복 수를 조절할 수 있어야 한다.
- 반복 결과는 미리보기와 다운로드에 반영되어야 한다.

### 4-12. 미리보기
- 새 창에서 미리보기를 열 수 있어야 한다.
- 숫자값이 보이는 패턴 그리드가 표시되어야 한다.

### 4-13. 다운로드
- 다음 포맷 다운로드를 지원해야 한다.
- CSV
- XLSX
- PDF

### 4-14. Undo / Redo
- 편집 영역에서 undo/redo를 지원해야 한다.
- 버튼 UI와 단축키를 모두 제공해야 한다.
- 단축키:
  - `Cmd+Z / Ctrl+Z`
  - `Shift+Cmd+Z / Ctrl+Y`

## 5. 핵심 용어 정의
- `셀 색(0)`: 바탕 셀 색
- `메인 실 색(1)`: 메인 실 영역 색
- `원본 패턴`: 사용자가 직접 편집하는 기본 패턴
- `반복 패턴`: 원본 패턴을 확장한 최종 출력 결과
- `요크 패턴 줄임`: 뜨개 세로 나눗셈 계산을 이용한 자동 형태 생성

## 6. 비기능 요구사항
- 단일 페이지 내에서 흐름이 자연스러워야 한다.
- 입력 단계와 편집 단계가 명확히 구분되어야 한다.
- 편집 영역이 가장 중요한 작업 공간으로 보이도록 해야 한다.
- 많은 기능이 있어도 복잡해 보이지 않아야 한다.
- 데스크톱 우선 UX를 기준으로 설계하되, 좁은 화면에서도 깨지지 않아야 한다.

## 7. 현재 반드시 유지해야 하는 사항
- 기존 기능 삭제 금지
- 기존 계산 로직 변경 금지
- 세로 나눗셈 계산 방식 유지
- `0`은 바탕값, `1`은 메인 실 색 의미 유지
- 그리드 먼저 생성 후 편집하는 흐름 유지
- 이미지/CSV/XLSX 불러오기 유지
- 편집 영역에서 직접 수정 가능한 구조 유지
- 반복 패턴, 미리보기, 다운로드 유지
- undo/redo 유지

## 8. 현재 UX에서 특히 개선이 필요한 영역
- 그리드 설정 완료 후 편집 영역 진입 경험
- 편집 도구와 출력 도구의 우선순위 구분
- 색상 설정 UI의 명확성
- 파일 불러오기 적용 규칙의 가시성
- 편집 영역 상단 액션 정리
- 반복 패턴과 출력 단계의 단계감 표현

## 9. stitch에게 요구할 사항 리스트
- 현재 기능은 삭제하지 말 것
- 계산 로직은 건드리지 말 것
- UI/UX 개선 중심으로 제안할 것
- 단일 페이지 구조를 유지하되 정보 구조를 더 명확히 정리할 것
- 입력 단계와 편집 단계를 시각적으로 분리할 것
- 그리드 생성 이후 편집 영역을 가장 중요한 작업 구간으로 설계할 것
- 편집 영역 상단 액션을 역할별로 그룹핑할 것
- 색상 설정 UI를 더 직관적인 팔레트 구조로 개선할 것
- `셀 색(0)`과 `메인 실 색(1)`의 역할 차이를 시각적으로 분명히 보여줄 것
- 파일 불러오기 버튼의 역할을 명확히 보여줄 것
- 이미지/CSV/XLSX가 현재 그리드에 적용된다는 점을 UX로 설명할 것
- 반복 패턴과 다운로드는 편집 이후 단계처럼 보이게 정리할 것
- 전체적으로 뜨개 작업 도구처럼 보이도록, 너무 일반적인 폼 UI처럼 보이지 않게 설계할 것
- 실사용자가 “다음에 뭘 해야 하는지” 바로 이해할 수 있게 흐름을 설계할 것

## 10. stitch 산출물 요청 형태
- 데스크톱 기준 메인 화면 디자인
- 입력 전/입력 완료 후 상태를 포함한 상태별 화면
- 그리드 생성 전 / 생성 후 편집 상태를 구분한 디자인
- 편집 영역 중심 UI 제안
- 주요 버튼/패널 우선순위가 드러나는 정보 구조 제안
